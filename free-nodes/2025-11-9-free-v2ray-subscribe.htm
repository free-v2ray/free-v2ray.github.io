确保您已经在App Store下载并安装了最新版的小火箭应用。打开应用后，您将看到简洁的主界面，顶部是代理开关，下方是节点列表和流量统计。

  1. 点击右上角的"+"按钮，选择"添加配置"

    • 选择合适的加密方式（通常为auto或aes-128-gcm）
    • 根据节点设置选择传输协议和伪装类型

- 路由规则：可以设置特定网站或IP段走直连或代理

第五步：保存并启用配置

填写完所有必要信息后，点击右上角的"完成"按钮保存配置。返回主界面后，找到您刚添加的节点，点击右侧的开关按钮即可启用连接。

常见问题排查与优化建议

即使按照上述步骤操作，有时仍可能遇到连接问题。以下是几个常见问题及解决方法：

连接失败的可能原因

  1. 节点信息错误：仔细检查服务器地址、端口、UUID等关键信息是否填写正确
  2. 网络环境限制：某些公共网络（如公司、学校）可能会主动拦截代理流量
  3. 节点失效：服务商可能已更新节点配置，需要获取最新信息
  4. 时间不同步：V2ray对时间同步要求严格，确保设备时间准确

提升连接速度的技巧

  1. 选择地理位置近的节点：物理距离越近，延迟通常越低
  2. 尝试不同传输协议：WebSocket在某些网络环境下可能比TCP更快
  3. 关闭IPv6：在iOS设置中禁用IPv6可能改善某些网络下的连接质量
  4. 调整加密方式：轻量级加密（如chacha20）可能比aes-gcm更快

安全使用建议

  1. 定期更新节点配置，避免长期使用同一节点
  2. 不要随意使用来源不明的免费节点
  3. 在公共场合使用代理时，注意保护个人隐私
  4. 考虑配合Clash等工具实现更精细的流量控制

2. 在小火箭中点击"配置"→"订阅"→"添加订阅"
3. 粘贴订阅链接并设置自动更新频率
4. 保存后，小火箭会自动获取并更新所有节点

订阅功能不仅节省了手动配置的时间，还能确保您始终使用最新的节点信息，大大提升了使用体验。

FAQ

Matsuri 节点延迟排序功能如何使用？
Matsuri 会定期测速节点延迟，并根据结果排序。用户可选择延迟最低的节点进行访问，结合策略组实现自动切换，提高网络速度和连接稳定性。
Nekobox 可以定时更新订阅节点吗？
支持。用户可设置定时更新频率，例如每天或每小时，程序会自动下载订阅节点并替换旧节点，确保节点始终可用并保持最新线路信息。
如何在 Nginx 上为 Vmess/WS 配置反向代理与 TLS？
在 Nginx 中配置 HTTPS 站点并将特定路径（如 /ray）反向代理到 V2Ray 的 WebSocket 端口，确保 proxy_set_header 设置正确（Host、X-Real-IP 等）。同时使用 certbot 获取并自动续期 TLS 证书以维护安全。
VLESS + XTLS 如何在高延迟网络中优化连接？
XTLS 模式通过减少握手和加密开销降低延迟。结合策略组和多节点切换，可在高延迟或不稳定网络环境下保证网页浏览、视频播放和游戏连接稳定，提高整体体验。
Mellow 与 Clash 的核心区别是什么？
Mellow 采用基于透明代理的架构，适合路由或系统级流量重定向，而 Clash 更偏向规则式应用代理。Mellow 的灵活性更高，但配置复杂；Clash 则兼顾易用性与功能扩展性。
WinXray 的多出口节点如何优化访问速度？
通过为不同流量类型设置不同出口节点，WinXray 可以自动选择延迟最低的节点处理请求。结合策略组，可提高网页访问、视频播放和游戏等应用的速度与稳定性。
何时应选择 VLess 而非 Vmess？
若追求更轻量、高性能的连接且采用外部 TLS 或 XTLS 进行加密，建议选择 VLess。VLess 去除了内嵌加密，能更好配合现代传输（XTLS/Reality）实现更低延迟与更好扩展性。
Nekoray 支持节点批量更新吗？
支持。Nekoray 可通过订阅 URL 批量下载最新节点并替换旧节点。结合定时更新功能，可保持节点列表最新，减少手动维护和配置工作。
Surfboard 如何结合规则实现智能节点切换？
Surfboard 可根据规则文件和策略组自动选择延迟低、稳定性高的节点处理流量。不同应用流量可分配不同节点，优化访问速度和稳定性，减少手动切换节点的操作。
Quantumult X 的自动策略组如何工作？
Quantumult X 的自动策略组可根据延迟、可用性或测速结果自动选择最优节点，用户无需手动切换。可在策略组设置中开启“延迟优先”或“可用性优先”模式以优化体验。
